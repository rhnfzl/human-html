Question

Which vector database fits a 1M-vector / sub-100ms / single-region / Python-first workload best for user-note semantic search, given that we already run Postgres in production?

Method

One week of investigation: read each vendor's docs, run a 100k-vector benchmark with our embedding model on a single instance, check data-residency options, estimate cost at 1M vectors and 200 queries/min.

Synthesis

Postgres + pgvector wins on three of four axes: lowest cost, data residency confirmed, no new vendor to onboard. Latency is acceptable (22ms vs Pinecone's 14ms; both well inside the 100ms target). The trade-off is operational: scaling vector indexes on a transactional Postgres needs care (separate read replica recommended at > 5M vectors).

For Q3 ship-by-July: pgvector on existing RDS. Revisit at 5M+ vectors or if p95 drifts past 60ms.

Open threads

  • Embedding model choice (text-embedding-3-small vs Cohere multilingual) is its own decision; not blocked by vector store choice.
  • Multi-region replication of vector indexes once we add a second region: pgvector handles via logical replication, but write throughput needs validation.
  • Privacy review of user notes (PII handling) before any of this ships.
