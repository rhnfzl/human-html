One-click "Generate a structured summary from a chat transcript" prototype

Kind prototype Created 2026-05-25 Owner Jordan Ellis Status Draft Read time ~6 min Source canonical-example

In plain terms

  • What this does for the user: A user who's already discussed a topic with a colleague in chat can hit one button to generate a draft summary from that conversation, instead of re-typing it into the structured form.
  • Why it matters: Colleagues describe requests in chat naturally; the gap between chat and the structured form is a friction point we've heard about three times in user research.
  • What's being asked: Play with the prototype, give thumbs-up / thumbs-down on the UX before we invest in productionising the LLM extraction step.

Goal

Learn whether users trust an LLM-extracted summary enough to edit it down rather than start blank. Three users in 30-minute sessions next week.

Current vs target

Today: user copies key points from chat into a 12-field form. Target: prototype "Generate from chat" button fills the form, user edits.

What's mocked

  • Mocked: LLM call - the prototype uses a hardcoded JSON response per fixture transcript. Real extraction lives in a follow-up sprint if this UX works.
  • Mocked: Confidence indicators per field - the prototype shows green/yellow indicators sampled from a uniform random instead of real model confidence.
  • Mocked: Edit history - the prototype doesn't persist edits; close the tab and they're gone.

What's real

  • Real: The full structured form rendered from existing production templates.
  • Real: The "Generate from chat" button placement in the chat sidebar - same component, same routing, same auth boundary.
  • Real: The post-generation diff view (showing which fields the prototype filled vs left blank).
