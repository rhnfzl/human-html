human-html

Review surfaces a teammate actually reads, not a Markdown wall that gets skimmed. One self-contained HTML page per plan, review, or decision, enforced by a validated content contract.

npx skills add rhnfzl/human-html

The nine kinds

plan

A forward-looking work plan with scope, milestones, risks, and a rollback path.

 review

A critique of a code change or PR, with findings ranked by severity.

 architecture

An explainer of how a system works, with diagrams and open questions.

 understanding

What the team learned tracing a flow end to end through the code.

 research

External evidence synthesized into an options comparison and a recommendation.

 decision

A locked choice laid out with options, trade-offs, and consequences.

 prototype

A live, interactive demo that still reads with JavaScript disabled.

 status

A snapshot of where work stands: progress, blockers, and next steps.

 incident

A postmortem with a timeline, impact, root cause, and corrective actions.
